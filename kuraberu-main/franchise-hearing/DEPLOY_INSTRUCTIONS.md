# 🚀 フランチャイズ登録システム デプロイ手順書

## 1. GAS WebApp のデプロイ

### Step 1: GASプロジェクトの準備
1. [Google Apps Script](https://script.google.com) にアクセス
2. 既存のプロジェクトを開く（notify.gsが含まれているプロジェクト）
3. notify.gsファイルに新しい加盟店登録システムのコードが追加されていることを確認

### Step 2: WebAppのデプロイ
1. GASプロジェクトで「デプロイ」→「新しいデプロイ」をクリック
2. デプロイ設定：
   - **種類**: ウェブアプリ
   - **説明**: 「フランチャイズ登録システム v1.0」
   - **実行ユーザー**: 自分
   - **アクセスできるユーザー**: 全員
3. 「デプロイ」ボタンをクリック
4. **重要**: 生成されたWebAppのURLをコピー保存

### Step 3: 権限の設定
1. 初回デプロイ時は認証が求められます
2. 「アクセス許可を確認」をクリック
3. Googleアカウントでログインし、権限を許可

## 2. フロントエンドの設定更新

### Step 4: API URL の更新
1. `config.js`ファイルを開く
2. `API_URL`を新しくデプロイしたWebAppのURLに更新：

```javascript
const FRANCHISE_CONFIG = {
    // 🔧 ここを新しいWebAppのURLに更新
    API_URL: 'https://script.google.com/macros/s/YOUR_NEW_WEBAPP_URL/exec',
    // ... その他の設定
};
```

### Step 5: 設定の確認
以下のファイルがconfig.jsを読み込んでいることを確認：
- ✅ `01_basic_info.html`
- ✅ `02_area_selection.html`  
- ✅ `03_confirmation.html`

## 3. テスト手順

### Step 6: 動作テスト
1. `01_basic_info.html`をブラウザで開く
2. フォームに必要事項を入力
3. エリア選択まで進む
4. 確認画面で「登録を送信する」をクリック
5. 正常に登録完了メッセージが表示されることを確認

### Step 7: スプレッドシート確認
1. プロパティで設定されたスプレッドシートを開く
2. 「加盟店登録」シートが自動作成されていることを確認
3. 登録データが正しく保存されていることを確認

## 4. トラブルシューティング

### よくある問題と解決方法

#### ❌ 「403 Forbidden」エラー
- **原因**: WebAppの権限設定が正しくない
- **解決方法**: 
  1. GASプロジェクトで「デプロイ」→「デプロイを管理」
  2. 「アクセスできるユーザー」を「全員」に変更
  3. 新しいバージョンでデプロイ

#### ❌ 「CORS エラー」
- **原因**: ブラウザのCORS制限
- **解決方法**: 
  1. ローカルファイルではなくWebサーバー経由でアクセス
  2. HTTPSでホスティング

#### ❌ API呼び出しがタイムアウト
- **原因**: GASの実行時間制限
- **解決方法**:
  1. `config.js`の`API_TIMEOUT`値を増加
  2. GASコードの最適化

#### ❌ スプレッドシートにデータが保存されない
- **原因**: SPREADSHEET_IDが正しく設定されていない
- **解決方法**:
  1. GASプロジェクトのプロパティを確認
  2. スプレッドシートIDが正しく設定されているかチェック
