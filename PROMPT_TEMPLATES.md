# Claude Code プロンプトテンプレート集

## 🎯 基本テンプレート
```bash
# コピペして使用
"""
@PROJECT_MAP.md と @CLAUDE.md を必ず読んでルールに従ってください。

【作業内容】
- プロジェクト: [franchise-dashboard/admin-dashboard/etc]
- ファイル: [パス/ファイル名]（新規作成/修正）
- 機能: [具体的な機能説明]

【制約条件】
1. 1,000行以内で実装
2. try-catch必須でエラーハンドリング
3. console.logでデバッグ情報出力
4. 他のファイルは一切触らない
5. フレームワーク使用禁止

【テスト方法】
- test/フォルダに単体テストファイル作成
- ブラウザのコンソールで動作確認

【完了条件】
- エラーなしで動作すること
- 既存機能に影響がないこと
"""
```

## 📋 具体例集

### 1. API機能追加
```bash
"""
@PROJECT_MAP.md を読んでから作業開始。

【作業内容】
- プロジェクト: franchise-dashboard
- ファイル: js/api/merchants.js（新規作成）
- 機能: 加盟店一覧取得API

【要件】
- GAS APIから加盟店データ取得
- エラー時は「データ取得に失敗しました」をalert表示
- レスポンスをJSONでコンソール出力
- 他のAPIファイルは参照しない

【制約】
- 500行以内
- async/await使用
- 10秒でタイムアウト
"""
```

### 2. UI機能追加
```bash
"""
@CLAUDE.md のルールを守って実装。

【作業内容】
- プロジェクト: admin-dashboard
- ファイル: js/components/merchant-table.js（新規）
- 機能: 加盟店一覧テーブル表示

【要件】
- HTMLテーブルを動的生成
- ソート機能（名前、登録日順）
- ページネーション（20件ずつ）
- レスポンシブ対応

【制約】
- jQueryやBootstrap禁止
- vanilla JavaScriptのみ
- 既存CSSを活用
"""
```

### 3. 修正作業
```bash
"""
エラーが発生。PROJECT_MAP.mdを確認してから修正。

【エラー内容】
- ファイル: admin-dashboard/index.html
- エラー: 「○○が定義されていません」
- 発生状況: ページ読み込み時

【修正方針】
- 最小限の修正に留める
- 動いている他の機能に影響させない
- 修正前後のテスト必須

【注意】
- 完成している機能は絶対触らない
- バックアップ取ってから作業
"""
```

## 🧪 テスト用テンプレート
```bash
"""
【テスト環境構築】
- ファイル: test/test-[機能名].html
- 目的: [機能名]の単体テスト

【テスト項目】
1. 正常ケース
2. エラーケース
3. 境界値テスト
4. 既存機能への影響確認

【テストコード例】
- ボタンクリックで関数実行
- 結果をコンソールとHTMLに表示
- エラー時の挙動確認
"""
```

## 🚨 緊急時テンプレート
```bash
"""
🚨緊急事態🚨

【状況】
- 機能: [壊れた機能名]
- 影響: [影響範囲]
- 緊急度: 高

【対応方針】
1. 壊れた部分のみ修正
2. 応急処置を優先
3. 完全修正は後日

【制約】
- 他の機能は絶対触らない
- 最小限の変更のみ
- 既存コードを最大限活用

EMERGENCY_MODE = true で新機能無効化
"""
```

## 💡 効率化のコツ
- 1つの機能につき1回のプロンプト
- 複雑な機能は段階に分けて依頼
- エラーが出たら即座にCLAUDE.mdの呪文を使用
- テストは必ず別ファイルで実行
