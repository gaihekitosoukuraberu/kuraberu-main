
## 🔧 技術制約
- **ファイルサイズ**: 1ファイル最大1,000行
- **フレームワーク**: 使用禁止（フレームワークレス）
- **エラー処理**: 必須（try-catch + console.log + alert）
- **API**: GAS経由でスプレッドシート操作
- **テスト**: test/フォルダで単体テスト実行

## 🚫 絶対やってはいけないこと
- 動いているコードの直接編集
- フレームワークの導入
- 大規模リファクタリング
- テストなしの本番デプロイ
- GASの構造変更

## ✅ 安全な開発手順
1. **現状把握**: PROJECT_MAP.mdとCLAUDE.mdを読む
2. **別ファイル作成**: test/や new-features/フォルダで開発
3. **単体テスト**: 各機能を個別にテスト
4. **段階的統合**: 少しずつ既存システムに統合
5. **動作確認**: 既存機能が壊れていないことを確認
