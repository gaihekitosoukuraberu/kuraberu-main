# エラー予防・対策ルール

## 🛡️ エラー回避の呪文
新機能追加時に必ず言う：
```
「以下の順番で作業して：
1. PROJECT_MAP.mdで現在動いてる機能を確認
2. 新機能を別ファイル（test/フォルダ）で作成
3. 既存機能と統合する前に単体テスト
4. 問題なければ段階的に統合
5. 統合後、全体の動作確認」
```

## 📝 ERROR_HISTORY.md（エラー履歴）
```markdown
# エラー履歴と対処法

## 2025/09/25
- **エラー**: Slack通知が来なくなった
- **原因**: GASファイルの修正により通知機能に影響
- **対処法**: GASのSlack API部分を確認、実行ログをチェック
- **予防策**: Slack関連機能は別ファイルに分離

## エラー対処テンプレート
1. エラーの詳細をコンソールで確認
2. 最後に変更したファイルを特定
3. バックアップから復旧（緊急時）
4. 原因調査と修正
5. テスト実行
6. 同じエラーの予防策を文書化
```
